# Study Planner - הנחיות לתלמידים

בתרגיל הזה תבנו אפליקציית Node.js קטנה בשם Study Planner.

האפליקציה תעזור לתלמיד לבנות תוכנית לימוד קצרה להיום: לטעון מידע על תלמיד, להציג משימות לימוד, לבנות תוכנית קצרה, ולהציג תזכורת.

המטרה היא לבנות פרויקט קטן עם משמעות אמיתית, ובתוכו להשתמש בחומר האסינכרוני שנלמד השבוע.

הפרויקט צריך לרוץ דרך:

```powershell
npm start
```

אין צורך להשתמש בספריות חיצוניות.

## קבצים נדרשים

צרו את הקבצים הבאים:

- `package.json`
- `data.js`
- `callbacks.js`
- `promises.js`
- `asyncAwait.js`
- `main.js`

לתוספת המומלצת של JSON אפשר להוסיף גם:

- `storage.js`
- `saved-plan.json`

## מה האפליקציה צריכה לעשות

כאשר מריצים את האפליקציה, היא צריכה להציג תהליך קצר של תכנון יום לימוד.

האפליקציה צריכה לכלול:

- פתיחה שמראה שהאפליקציה התחילה לעבוד.
- הצגה של מידע על תלמיד קיים.
- מצב שבו מחפשים תלמיד שלא קיים.
- בנייה של תוכנית לימוד מתוך כמה משימות.
- מצב שבו נבחרת משימת לימוד שלא קיימת.
- הצגה של תזכורת לימוד לתלמיד.
- מצב שבו אי אפשר להכין תזכורת בגלל שחסר מידע.

## נתונים שחייבים להיות בפרויקט

ב-`data.js` צרו מידע פשוט:

- לפחות שני תלמידים.
- לפחות תלמיד אחד עם פרטי קשר לתזכורת.
- לפחות תלמיד אחד בלי פרטי קשר לתזכורת.
- לפחות שלוש משימות לימוד.

כל משימת לימוד צריכה לכלול:

- שם או כותרת.
- זמן משוער בדקות.

## דרישות טכניות חובה

במהלך הפרויקט חובה להשתמש בכל הדברים הבאים:

- `setTimeout`
- לפחות callback אחד שמועבר כפרמטר לפונקציה.
- לפחות callback אחד בתבנית error-first:

```js
(error, result)
```

- בדיקת שגיאה לפני שימוש בתוצאה.
- לפחות Promise אחד.
- שימוש ב-`.then()`.
- שימוש ב-`.catch()`.
- שימוש ב-`.finally()`.
- שרשרת קצרה של Promises.
- לפחות פונקציית `async` אחת.
- שימוש ב-`await`.
- שימוש ב-`try/catch`.

הדרישות הטכניות חייבות להופיע בפרויקט, אבל אין חובה שכל פעולה באפליקציה תהיה כתובה בדרך אחת מסוימת. בחרו איפה לשלב כל נושא בצורה שנראית לכם הגיונית.

## תוספת מומלצת: שמירה וטעינה מקובץ JSON

מומלץ להוסיף לפרויקט שמירה וטעינה של מידע מקובץ JSON קטן.

לדוגמה, אפשר לשמור את תוכנית הלימוד האחרונה בקובץ `saved-plan.json`, ואז בתחילת הרצה לקרוא את הקובץ ולהציג סיכום קצר של התוכנית הקודמת.

התוספת הזאת לא צריכה להיות Database ולא צריכה להיות מערכת קבצים גדולה. מספיק קובץ JSON אחד עם מידע פשוט.

## מה הפלט צריך להראות

כאשר מריצים `npm start`, הפלט צריך להראות:

- שהאפליקציה התחילה לעבוד.
- שקוד רגיל ממשיך לרוץ גם כאשר פעולה אחרת מתעכבת.
- מידע על תלמיד שנמצא.
- הודעה ברורה עבור תלמיד שלא נמצא.
- תוכנית לימוד שנבנתה בהצלחה.
- הודעה ברורה עבור משימה שלא קיימת.
- תזכורת לימוד שהוכנה בהצלחה.
- הודעה ברורה עבור תזכורת שלא ניתן להכין.
- אם הוספתם את תוספת ה-JSON: סיכום של תוכנית שנקראה מקובץ והודעה שהתוכנית נשמרה.
